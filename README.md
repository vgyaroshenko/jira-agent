# Jira QA Agent

AI-помічник для QA-інженера: генерує тест-кейси, створює баг-репорти і перевіряє якість описання задач у Jira. Працює через **Claude Code** — окремий API-ключ Anthropic не потрібен.

---

## Як користуватись

1. Відкрий папку проекту в **Claude Code**
2. Активуй віртуальне середовище:
   ```bash
   source .venv/bin/activate
   ```
3. Напиши в чат що потрібно зробити — Claude сам запустить всі команди і запостить результат в Jira

**Приклади того що писати в чат:**

| Що хочу | Що написати Claude Code |
|---|---|
| Згенерувати тест-кейси | *"згенеруй тест-кейси для GN-1808"* |
| Створити баг-репорт | *"створи баг: при натисканні кнопки Оплатити виникає помилка 500, відтворюється на Chrome якщо в кошику більше 3 товарів"* |
| Створити задачу | *"зроби задачу в проект GN"* — Claude запитає деталі і мову |
| Перевірити якість задачі | *"перевір якість описання задачі GN-1808"* |
| Переглянути стан спринту | *"покажи стан спринту GN"* |

Команди нижче (`fetch`, `comment`, `new-bug`, `sprint`) — це інструменти які Claude запускає автоматично. Їх не потрібно виконувати вручну.

## Команди

### fetch — отримати задачу з Jira

```bash
python main.py fetch GN-1808
```

Виводить дані задачі в термінал: заголовок, опис, статус, критерії приймання, ID репортера.
Claude Code читає цей вивід і далі генерує тест-кейси або перевіряє якість.

---

### comment — додати коментар до задачі

```bash
# Простий коментар
echo "Текст коментаря" | python main.py comment GN-1808

# Багаторядковий текст
python main.py comment GN-1808 << 'EOF'
🤖 Тест-кейси:
TC-001 — ...
EOF

# З тегом користувача
python main.py comment GN-1808 --mention 557058:xxxxxxxx << 'EOF'
🤖 Будь ласка, доповни задачу:
- Відсутні кроки відтворення
EOF
```

| Параметр | Опис |
|---|---|
| `ISSUE_KEY` | Ключ задачі (обов'язково) |
| `--mention ID` | Account ID для тегу в коментарі |

---

### new-bug — створити баг в Jira

```bash
python main.py new-bug --title "Попап не відкривається на картці товару" --lang UA --related GN-1652 << 'EOF'
##ENV##
Середовище: PROD
Браузер: Chrome 124
Розділ: Медодяг → Dr. Gray
Сторінка: картка товару з кодом 38218

##DESC##
При натисканні кнопки «Таблиця розмірів» відкривається нова вкладка замість попапу.

##STEPS##
1. Відкрити картку товару з кодом 38218
2. Знайти кнопку «Таблиця розмірів»
3. Натиснути на кнопку

##EXPECTED##
- Таблиця розмірів відкривається у попапі на поточній сторінці

##ACTUAL##
- Відкривається нова вкладка браузера
- Таблиця розмірів не відображається

##ADDITIONAL##
Проблема лише для бренду Dr. Gray
EOF
```

Баг автоматично додається до активного спринту. Якщо спринту немає — потрапляє в беклог.

| Параметр | Опис |
|---|---|
| `--title TEXT` | Заголовок бага до 80 символів (обов'язково) |
| `--lang UA\|RU\|EN` | Мова баг-репорту (визначається автоматично якщо не вказати) |
| `--project KEY` | Ключ проекту, наприклад `GN` |
| `--related KEY` | Ключ пов'язаної задачі — баг створиться в тому ж проекті і буде прилінкований |

Секції опису (`##ENV##`, `##DESC##`, `##STEPS##`, `##EXPECTED##`, `##ACTUAL##`, `##ADDITIONAL##`) в Jira відображаються як окремі заголовки зі списками.

### new-task — створити задачу в Jira

```bash
python main.py new-task --title "Обмеження — реферальна акція і welcome bonus" --lang UA --project GN << 'EOF'
##DESC##
Необхідно реалізувати обмеження: реферальна акція та welcome bonus не можуть використовуватися одночасно в одному замовленні.

##WHAT##
1. Інформування на сторінках акцій\nЯкщо у користувача активні обидві акції — виводити попереджувальний текст у верхній частині опису.
2. Логіка застосування знижки\nПри додаванні товарів з обох акцій застосовується лише перша знижка. Товари з другої акції йдуть за повною вартістю.

##AC##
AC1. Перевірка доступності акцій\nОбидві акції активні у користувача.\n✅ Очікуваний результат: на сторінках акцій відображається попереджувальний текст.
AC2. Логіка знижки\nТовари з обох акцій додані в кошик.\n✅ Очікуваний результат: застосовується лише перша знижка, друга ігнорується.
EOF
```

| Параметр | Опис |
|---|---|
| `--title TEXT` | Заголовок задачі (обов'язково) |
| `--lang UA\|RU\|EN` | Мова задачі (визначається автоматично якщо не вказати) |
| `--project KEY` | Ключ проекту, наприклад `GN` (обов'язково) |
| `--type Story\|Task` | Тип задачі (за замовчуванням `Story`) |

В Jira задача створюється з секціями: опис, "Що потрібно реалізувати" (нумерований список), "Критерії приймання" (буллет-список). Символ `\n` у ##WHAT## і ##AC## розбиває текст на окремі рядки всередині пункту.

---

### sprint — переглянути стан спринту

```bash
python main.py sprint GN
```

Виводить задачі активного спринту з позначками:

```
Знайдено задач: 12

GN-1800  [In Progress]  ❌TC  ❌QC  —  Назва задачі
GN-1801  [In Progress]  ✅TC  ✅QC  —  Інша задача
GN-1802  [Review]       ❌TC  ✅QC  —  Ще одна задача
```

| Позначка | Значення |
|---|---|
| `✅TC` | Тест-кейси вже є |
| `❌TC` | Тест-кейсів немає |
| `✅QC` | Коментар про якість є |
| `❌QC` | Коментаря немає |

Задачі зі статусами `QA`, `Test`, `Ready for Deploy`, `QA Prod`, `Done` не виводяться.

## Встановлення

```bash
# 1. Клонувати репозиторій
git clone <repo-url>
cd jira-agent

# 2. Створити віртуальне середовище
python -m venv .venv
source .venv/bin/activate      # macOS / Linux
.venv\Scripts\activate         # Windows

# 3. Встановити залежності
pip install -r requirements.txt

# 4. Скопіювати файл налаштувань
cp .env.example .env
```

Відкрити `.env` і заповнити:

```env
JIRA_URL=https://yourcompany.atlassian.net
JIRA_EMAIL=your@email.com
JIRA_API_TOKEN=your_jira_api_token
JIRA_PROJECT_KEY=GN,CCW,DEV
```

**JIRA_API_TOKEN** — Jira → Profile → Security → API tokens
